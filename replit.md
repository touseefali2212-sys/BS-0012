# NetSphere - Enterprise ISP Billing & Management System

## Overview
NetSphere is a multi-tenant SaaS ISP Billing & Operations Management System designed for Internet Service Providers. It offers a comprehensive suite of 29+ modules, each with expandable submenus and tab-based sub-features, to manage billing, operations, customer relations, and network infrastructure. The system is built to be enterprise-grade, supporting various ISP operational needs from customer management to network monitoring and financial accounting.

## System Architecture

### UI/UX
The frontend is built with React 18, Vite, Tailwind CSS, shadcn/ui components, wouter for routing, TanStack React Query, Leaflet maps for GIS features, Recharts for data visualization, and Framer Motion for animations. Design principles emphasize an enterprise blue theme with consistent elevation utilities. Dynamic theming (dark/light) is supported. Many modules feature gradient-themed KPI cards and dark-header tables for a modern, professional aesthetic.

### Technical Implementations
- **Frontend**: Utilizes React hooks (`use-auth`, `use-tab`, `use-toast`, `use-mobile`, `use-permissions`) for state management and common functionalities.
- **Backend**: Developed with Express 5 and TypeScript (tsx). Drizzle ORM is used for database interactions, with `drizzle-kit` for schema management. Session-based authentication is implemented. `multer` handles file uploads.
- **Database**: PostgreSQL is the primary database, managed via Drizzle ORM.
- **Shared Components**: `shared/` directory houses common types and Zod validation schemas for data consistency between frontend and backend. `client/src/components/cir-customer-form-fields.tsx` and `client/src/components/corporate-customer-form-fields.tsx` are shared form field components.
- **Edit Profile Flow**: CIR/Corporate "Edit Profile" (from list page actions or profile page "Edit Profile" button) navigates to the same `/add-customer` page in edit mode via URL params `?type=corporate&edit=<id>` or `?type=cir&edit=<id>`. The Add Customer page detects edit mode, fetches existing data, pre-fills forms, uses PATCH instead of POST on save, and shows "Update" instead of "Save" buttons. This ensures the Edit and Add forms are always identical — same full-page layout with all tabs and fields.
- **ONT/ONU Integration with OLT**: All three customer profile pages (Normal, CIR, Corporate) include an "ONT/ONU Integration with OLT" section in the Network & Infrastructure tab. Shows linked ONU/ONT devices with full chain info (ONU → Splitter → OLT). Features: assign available ONU devices, unlink devices, view optical power/serial/MAC/IP, splitter details (ID, name, split ratio, PON port), and OLT details (ID, name, IP, vendor/model, port utilization, status). Schema: `onu_devices` table has `customerType` field to distinguish normal/cir/corporate customers. API: `GET /api/customers/:id/ont-integration?customerType=<type>`, `POST /api/customers/:id/ont-integration` (with customerType in body), `DELETE /api/customers/:id/ont-integration/:onuId?customerType=<type>`. Server enforces ownership checks on unlink and prevents double-assignment.
    - **OLT Management Module**: Dedicated OLT Management pages at `/olt-management` (listing) and `/olt-management/:id` (9-tab management). Listing page (`olt-list.tsx`) shows KPI cards (Total OLTs, Active, Total ONUs, Online ONUs), searchable OLT table with PON port utilization, ONU counts, status badges, and Add OLT dialog. Management page (`olt-management.tsx`) has 9 tabs: (1) OLT Overview - summary cards, device info, system health with live CPU/Memory/Temperature bars, recent alarms; (2) SNMP Monitoring - config display, live metric cards (CPU, Memory, Temp, Uptime, Voltage, Fan, Power), 24h performance bar chart; (3) PON Ports - expandable table showing all PON ports with connected ONUs, utilization, signal strength, reset actions; (4) ONU/ONT Devices - searchable table with activate/deactivate/reboot/view actions, export button; (5) Uplinks & Interfaces - table showing GE/10GE interfaces with traffic in/out, per-uplink traffic cards; (6) Traffic & Bandwidth - 24h traffic chart, peak/avg metrics, per-PON port bandwidth; (7) Alarms & Events - severity-tagged alarm table (Critical/Warning/Info) with acknowledge status; (8) Configuration - edit OLT settings (name, IP, vendor, model, ports, status, notes), SNMP credentials display, vendor profile cards (Huawei/ZTE/FiberHome/Nokia/VSOL/HSGQ); (9) Logs & History - activity log table with timestamps, actions, users, details. Uses simulated live metrics (5s refresh), dark gradient header theme, and integrates with existing OLT/ONU/Splitter APIs.
- **Core Modules**:
    - **Asset Management**: Comprehensive tracking of asset types, lists, transfers, assignments, allocation, and requests with detailed forms, analytics, and lifecycle tracking. Includes dedicated pages for Asset Tracking, Allocation, and Requests.
    - **Inventory & Product Management**: Master classification system for products, brand management, inventory list with real-time stock visibility across warehouses, batch and serial number tracking.
    - **Purchase Orders**: Full procurement workflow including creation, approval, receiving (GRN), and financial tracking.
    - **Customer Management**: Dedicated modules for general customers, CIR (Committed Information Rate), and Corporate customers, each with tailored management interfaces, mapping, and billing. Includes dynamic customer type management and customer rights configuration. Customer Profile features 14 tabs in order: Personal Information, Documents, Service Information, Network & Infrastructure, Service Scheduler, Customer Change Log (audit trail), Customer Enable/Disable Log (status change history), Product & Service Sales Invoices, Generated & Updated Bill/Invoices, Received Bill History, Remarks History, Support & Ticket History, Email/Message History, and Referrals. Sidebar buttons include Status Scheduler, Send Email/Message, Package Scheduler, and Package Change Request. Backend endpoints: `/api/customers/:id/audit-logs`, `/api/customers/:id/enable-disable-logs`, `/api/customers/:id/message-history` provide customer-specific filtered data from audit_logs, notification_dispatches, and message_logs tables. Customer List action buttons include: **Send SMS** (opens multi-channel notification dialog with In-App, SMS, WhatsApp, Email types and message categories like Bill Reminder, Invoice Softcopy, Account Suspend, Payment Confirmation, Service Activation, Package Change, Maintenance Notice, Welcome Message, and Custom), and **Active/Closed** (opens status toggle dialog with Active/Closed/Suspended/Expired options, reason field, and creates audit log entry for history tracking in Customer Change Log tab).
    - **Package Change & Bandwidth Management**: Full workflow page at `/package-change` with 7 tabs (New Request, Pending Approval, Approved, Implementation Queue, Completed, Rejected, History). Supports all customer types (Normal, CIR, Corporate) with customer search, billing preview calculator, approve/reject/implement dialogs, and audit trail. Schema: `packageChangeRequests` table. API: GET/POST/PATCH/DELETE `/api/package-change-requests`. Auto-integrates with Service Scheduler (creates entry) and Bandwidth History (for CIR/Corporate). Accessible from sidebar under Customers and via "Package Change Request" button on all customer profile pages.
    - **Service Scheduler Management**: Centralized service scheduling page at `/service-scheduler` with 7 workflow tabs (New Request, Pending, Approved, In Progress, Completed, Rejected, All History). Supports all customer types with customer search, 11 service types (package_upgrade/downgrade, installation, maintenance, equipment_new/replace, relocation, troubleshooting, disconnection, reconnection, other), equipment type selection, package selection for Normal upgrades/downgrades, date/time/assignee/priority fields, KPI cards, details dialog with status transitions. Schema: `serviceSchedulerRequests` table (includes customerName, customerType, scheduledDate, assignedTo, isUrgent). API: GET/POST/PATCH/DELETE `/api/service-scheduler-requests` with field allowlist on PATCH. Accessible from sidebar under Customers.
    - **Reseller Management**: Full reseller lifecycle management with a dedicated profile page (`/resellers/:id`) modeled after the CIR Customer Profile. Features a dark sidebar with key metrics, two-row tabbed content area (Personal Information, Network & Service, Billing & Finance, Commission & Bank, Agreement & Settings, Vendor Panels, Wallet Transactions, Customer Summary, Complain History, SMS History, Referrals), and a multi-section edit dialog covering all reseller fields including vendor panel configuration.
    - **HR & Payroll**: Employee management, attendance with break tracking, holiday/leave management, advance/loan management with EMI calculation, detailed salary processing with partial payment and payslip generation, bonus and commission management (including auto-commissions), employee types/roles, and shift scheduling.
    - **Accounting & Finance**: General ledger with hierarchical account types, income/expense entry, transaction management with various collection types (customer, recovery officer, CIR, corporate, reseller), refund/credit management, internal fund transfers, and a robust approval workflow system. Budget allocation is also supported.
    - **Network Operations**: IPAM (IP Address Management) with IP pool, subnet, VLAN management, and conflict detection. MikroTik/BRAS integration for device management, RADIUS/AAA for authentication/accounting, and bandwidth usage monitoring. Includes FTTH/P2P mapping for physical network infrastructure visualization.
    - **Notifications & Communication**: Centralized notification type management, SMS/Email/WhatsApp API integration for multi-channel communication, bulk campaign management, push SMS console, and push notification system.
    - **Reporting & Analytics**: A dedicated reports module with a dashboard and specific reports for customers, billing, payments, network, inventory, assets, HR, notifications, activity logs, and vendors.
    - **System Administration**: General settings (company profile, localization, financial, operational, branding), activity logging, payment gateway settings with webhooks and settlements, invoice template designer, HRM rights setup, and staff user login/account management.

### Feature Specifications
- **Data Grids**: Extensive use of searchable, filterable, and sortable data grids with pagination and bulk actions.
- **Forms**: Multi-step and multi-section forms with Zod validation, auto-fill capabilities, and conditional logic.
- **Analytics**: Numerous KPI cards, pie charts, bar charts, line charts, and area charts for data visualization and trend analysis across all modules.
- **Workflow Automation**: Features like auto-provisioning, auto-adjustment of receivables, automated notifications, and multi-level approval processes.
- **Security**: Role-based access control, session-based authentication, password policies, login activity logging, and audit trails for critical operations.
- **Mapping**: Interactive Leaflet maps for customer locations, network devices, and fiber infrastructure.
