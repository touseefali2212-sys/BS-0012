➕ 3️⃣ Approval Configuration Settings

Allow system admin to define:

A. Approval Levels

Level 1 – Accounts Officer
Level 2 – Finance Manager
Level 3 – Admin / Director

B. Threshold Rules

Example:

• Refund > 50,000 → Finance Manager
• Transfer > 200,000 → Admin
• Write-off > 100,000 → Director
• Wallet Adjustment > 10,000 → Manager

System should auto-route requests based on:

✔ Amount
✔ Branch
✔ Transaction Type
✔ Risk Category

C. Branch-Based Approval

Enable:

• Branch Manager approval
• Head Office approval
• Multi-branch routing

🔄 System Logic

When transaction requires approval:

✔ Mark status as Pending
✔ Lock transaction editing
✔ Send notification to approver
✔ Log request

If approved:

✔ Execute transaction
✔ Post accounting entry
✔ Update dashboard
✔ Unlock related modules

If rejected:

✔ Mark as Rejected
✔ Notify creator
✔ Allow correction

If escalated:

✔ Move to higher approval level
✔ Notify senior authority

🔐 Role-Based Access

Admin:
Full control

Finance Manager:
Approve high-value

Accounts Officer:
Create requests

Branch Manager:
Branch-level approvals

Viewer:
Read-only

No one can self-approve their own transaction.
