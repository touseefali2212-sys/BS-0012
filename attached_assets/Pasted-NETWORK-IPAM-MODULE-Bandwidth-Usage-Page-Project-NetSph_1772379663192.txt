NETWORK & IPAM MODULE
Bandwidth Usage Page

Project: NetSphere ISP Billing & Enterprise ERP

🎯 Objective

Develop a real-time and historical Bandwidth Usage Monitoring Page to track, analyze, and manage network traffic across customers, routers, VLANs, and POPs.

This page enables:

✔ Real-time traffic visibility
✔ Per-user bandwidth tracking
✔ POP-level usage analysis
✔ Peak hour monitoring
✔ Fair Usage Policy enforcement
✔ Capacity planning insights
✔ Revenue-based traffic analysis

This is your ISP performance control center.

6️⃣ Alerts & Fair Usage Monitoring

Automated monitoring system:

Alerts when:

✔ Router utilization > 80%
✔ Customer exceeds FUP limit
✔ Abnormal traffic spike
✔ Possible DDoS activity
✔ Interface down

Fair Usage Logic:

When customer exceeds data limit:

✔ Apply speed throttle
✔ Notify customer via SMS/Email
✔ Log FUP action
✔ Allow manual override

Escalation:

✔ Notify NOC team
✔ Create automatic task in Task Module

🔄 Automation Logic

When customer connects:

✔ Start session tracking
✔ Log upload/download

When traffic exceeds threshold:

✔ Trigger alert
✔ Log abnormal event

When monthly reset date reached:

✔ Reset usage counter
✔ Remove throttle (if applied)

When package upgraded:

✔ Update bandwidth monitoring profile
