ASSETS MODULE
Asset Assignment to Customer Page

Project: NetSphere ISP Billing & Enterprise ERP

🎯 Objective

Develop a centralized Asset Assignment to Customer Page to manage the allocation of physical and network assets (ONU, Router, CPE, Modem, IP Address, etc.) to customers.

This page ensures:

✔ Proper device allocation
✔ Real-time asset tracking
✔ Service-to-device linkage
✔ Prevention of duplicate assignment
✔ Revenue-linked asset tracking
✔ Clear customer-device ownership history

This page bridges Infrastructure with Billing & CRM.

Professional controlled assignment system.

Fields:

Assignment ID (Auto-generated)

Customer ID

Service ID

Asset Type (ONU / Router / CPE / Switch / Static IP)

VLAN (Auto-assign if required)

Installation Date

Assigned Technician

Security Deposit Amount

Rental or One-Time Sale

Device Condition (New / Used / Refurbished)

Notes

Options:

✔ Auto-Provision via MikroTik / BRAS
✔ Send Configuration to Device
✔ Notify Customer
✔ Generate Installation Invoice

Validation Rules:

❌ Cannot assign already assigned asset
❌ Cannot assign inactive inventory
❌ Must link to active service

🔄 Automation Logic

When asset assigned:

✔ Change inventory status to “Assigned”
✔ Update asset location to Customer
✔ Link IP & VLAN
✔ Trigger provisioning
✔ Generate invoice (if required)

When service suspended:

✔ Auto disable device
✔ Update assignment status

When device returned:

✔ Remove IP allocation
✔ Clear VLAN mapping
✔ Mark inventory as “Available”
✔ Log condition

If device marked faulty:

✔ Auto-create maintenance ticket
✔ Move to Repair Queue
