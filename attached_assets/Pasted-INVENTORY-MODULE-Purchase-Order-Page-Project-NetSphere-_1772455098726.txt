INVENTORY MODULE
Purchase Order Page

Project: NetSphere ISP Billing & Enterprise ERP

🎯 Objective

Develop a structured Purchase Order (PO) Page to manage, approve, and track all procurement activities from suppliers.

This page ensures:

✔ Controlled purchasing process
✔ Cost transparency
✔ Supplier accountability
✔ Approval governance
✔ Accurate stock planning
✔ Financial integration

It serves as the bridge between Procurement, Inventory, and Accounts Payable.

📊 Page Layout Structure

The page contains 5 core sections:

1️⃣ Purchase Order Overview Dashboard
2️⃣ Create / Edit Purchase Order
3️⃣ Purchase Order Master List
4️⃣ PO Details & Receiving Panel
5️⃣ Financial & Payment Tracking Panel

1️⃣ Purchase Order Overview Dashboard

KPI Cards:

Pending Approvals

Outstanding PO Value

Analytics:

📉 Price Variation Trend

2️⃣ Create / Edit Purchase Order

Professional structured PO form.

Header Section:

PO Number (Auto-generated)

Supplier (Linked to Suppliers Page)

Warehouse Destination

PO Date

Expected Delivery Date

Payment Terms

Currency

Linked Allocation / Request ID (if any)

Priority Level

Approval Required (Yes/No)

Line Item Section:

For each product:

Product (Linked to Brands & Products)

Description

Quantity

Unit Price

Discount (%)

Tax (%)

Subtotal

Total

System auto-calculates:

✔ Line totals
✔ Tax amount
✔ Grand total
✔ Estimated inventory value

Footer Section:

Shipping Cost

Additional Charges

Notes

Attachments (Quotation, Contract)

Validation Rules:

❌ Cannot exceed supplier credit limit
❌ Cannot create PO for inactive supplier
❌ Unit price variance alert if exceeds previous average

4️⃣ PO Details & Receiving Panel

Once goods arrive:

Receiving Workflow:

Approved PO →
Create Goods Receipt Note (GRN) →
Verify quantities →
Update stock

Receiving Data:

Received Quantity

Damaged Quantity

Short Quantity

Serial Numbers (if applicable)

Warehouse Location

Received By

Inspection Status

System Updates:

✔ Stock increases
✔ Reserved stock converted
✔ Asset serials registered
✔ Update PO status (Partial / Full)

5️⃣ Financial & Payment Tracking Panel

Integrated with Accounts Payable.

Displays:

Total PO Value

Invoice Value

Amount Paid

Outstanding Balance

Payment Due Date

Payment Status (Unpaid / Partial / Paid)

Automation:

If payment recorded:

✔ Update supplier balance
✔ Update payable ledger

If credit limit exceeded:

✔ Block further POs
✔ Notify finance

🔄 Automation Logic

When PO approved:

✔ Reserve expected stock
✔ Notify warehouse

When GRN created:

✔ Update inventory levels
✔ Register serial numbers
✔ Link asset records (if applicable)

When fully received:

✔ Change status to Completed

If cancelled:

✔ Release reserved allocation

If price deviation > threshold:

✔ Alert procurement manager

🔐 Role-Based Access

Admin:
Full control

Procurement Manager:
Create & approve PO

Inventory Manager:

Finance Manager:
Manage payments

Department Manager:

Viewer:
Read-only

Only Admin can override approval hierarchy.
