SETTINGS MODULE
HRM Rights Setup Page

(App & Web Permissions – Module & Submenu Level Control)

Project: NetSphere ISP Billing & Enterprise ERP

🎯 Objective

Develop a centralized HRM Rights Setup Page to configure and manage role-based access permissions for all modules and submenus across both Web and Mobile App platforms.

This page ensures:

✔ Controlled system access
✔ Role-based permission governance
✔ Web vs App access separation
✔ Action-level control (View, Create, Edit, Delete, Approve)
✔ Secure data isolation
✔ Compliance-ready access management

It serves as the core access control framework (RBAC Engine) of the ERP.

3️⃣ Action-Level Permission Configuration

Advanced permission logic.

Granular Controls:

• View Own Data Only
• View Department Data
• View All Data
• Edit Own Records Only
• Approve Up To Specific Amount
• Restrict Deletion
• Restrict Financial Fields
• Restrict Cost Visibility

Example:

Sales Manager:
✔ Create Invoice
✔ Edit Invoice
❌ Delete Invoice
✔ View All Sales
❌ Edit Cost Price

Inventory Manager:
✔ Adjust Stock
✔ Transfer
❌ Approve Purchase Order

Conditional Access:

IF user belongs to Warehouse A
→ Can only view Warehouse A stock

4️⃣ App vs Web Access Control Panel

Separate toggles for:

Web Access
Mobile App Access

For each module/submenu:

✔ Web Only
✔ App Only
✔ Both
❌ Disabled

Example:

Technician Role:
✔ App: Assets Tracking
✔ App: Customer Map
❌ Web: Finance Module

Finance Role:
✔ Web: Sales & Reports
❌ App: Inventory Adjustment

Supports platform-specific UI restrictions.

🔄 Permission Inheritance Logic

Supports:

✔ Role-Based Access Control (RBAC)
✔ Permission Inheritance
✔ Parent-Child Module Control
✔ Override Rules

Example:

If Inventory Module disabled →
All submenus auto disabled

If Admin role →
Full access auto-granted

System prevents conflicting permissions.

🔐 Security Features

✔ Audit log of permission changes
✔ Track who modified role access
✔ Timestamp tracking
✔ Multi-factor confirmation for critical changes
✔ Lock system roles (Admin cannot be deleted)

Prevents:

• Unauthorized access
• Data leaks
• Accidental deletion
• Financial manipulation

📊 Permission Summary & Validation

Before saving changes:

System displays summary:

Total Permissions Granted

High-Risk Permissions

Financial Access Level

Delete Rights Enabled

Validation alerts if:

• Too many critical rights assigned
• Financial module given to non-finance role
• App-only user given restricted backend access
