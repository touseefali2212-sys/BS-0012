Customers Module
Package Change & Bandwidth Upgrade/Downgrade Page
(For Normal, Corporate & CIR Customers)
🎯 Objective

Develop a dedicated Package Change Management Page where users can:

✔ Request package upgrade or downgrade
✔ Handle Corporate & CIR custom plans
✔ Approve or reject requests
✔ Automatically update billing
✔ Sync with network (Radius / MikroTik / BRAS)
✔ Maintain full audit trail

This page should act as a controlled workflow system, not just a simple edit form.

🧩 Page Structure

Use a tab-based interface:

Tabs:
New Request
Pending Approval
Approved Requests
Implementation Queue
Completed Requests
Rejected Requests
Request History

Form to create upgrade/downgrade request.

CIR

Change Type:
Upgrade
Downgrade

New Bandwidth (for CIR/Corporate custom input)
Effective Date:
Immediate
Next Billing Cycle
Custom Date
Billing Preview (Auto Calculated):

✔ New Package Price
✔ Prorated Charges
✔ Adjustment Amount
✔ Tax Impact
✔ Final Bill Difference

Additional Options:
Reason for Change
Attach Supporting Documents
Urgent Request Toggle
Actions:

✔ Submit Request
✔ Save Draft

✔ Verify billing impact
✔ Validate package availability
✔ Check outstanding dues
