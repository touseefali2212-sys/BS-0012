ASSETS MODULE
Request & Approvals Page

Project: NetSphere ISP Billing & Enterprise ERP

🎯 Objective

Develop a centralized Request & Approvals Page to manage, review, and authorize all asset-related requests across the organization.

This page governs:

✔ Asset issuance
✔ Asset transfer
✔ Device replacement
✔ Maintenance requests
✔ Asset return
✔ Asset write-off
✔ Emergency equipment allocation

The goal is to enforce:

✔ Controlled asset flow
✔ Role-based approval
✔ Transparent accountability
✔ Complete audit compliance

4️⃣ Approval Workflow Panel

Multi-level approval system based on:

• Asset value
• Request type
• Department
• Financial impact

Example Workflow:

Technician →
Inventory Manager →
Network Manager →
Finance Manager (if value threshold exceeded) →
Admin Final Approval

Approval Actions:

✔ Approve
✔ Reject (Reason Required)
✔ Request More Information
✔ Escalate
✔ Reassign Approver

Each approval step records:

Approver Name

Date & Time

Decision

Comments

Status Flow:

Draft → Pending → Under Review → Approved → Executed → Closed

If rejected:

✔ Mandatory rejection reason
✔ Notification to requester
✔ Permanent log entry

5️⃣ Audit & Decision Log

Timeline Example:

🟢 12 Feb – Executed & Asset Assigned

Audit Log Includes:

Policy reference (if applicable)

This section cannot be edited or deleted.

🔄 Automation Logic

When request approved:

✔ Trigger related workflow (transfer / assignment / maintenance)
✔ Update asset status
✔ Notify responsible parties
✔ Create task (if required)
✔ Log financial impact

When request rejected:

✔ Unlock reserved asset
✔ Notify requester
✔ Archive request

If SLA exceeded:

✔ Auto-escalate
✔ Send alert to admin

🔐 Role-Based Access

Admin:
Full approval authority

Inventory Manager:
Approve asset movement

Network Manager:
Approve device allocation

Finance Manager:
Approve high-value assets

Department Head:
Approve department-level requests

Technician:
Submit request only

Viewer:
Read-only

Only Admin can override multi-level workflow.
